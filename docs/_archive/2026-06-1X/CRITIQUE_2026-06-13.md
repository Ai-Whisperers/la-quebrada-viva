# LQV Critique — 2026-06-13

Read-only audit. T-14 from escritura 2026-06-27. Renderer is byte-frozen at `85e86aa`; nothing below proposes touching `build_scene.py`.

## P0 — ship-blocking for escritura 2026-06-27

- `lqv/typologies/__init__.py:21` — `TYPOLOGIES` tuple lists `'cob_bottle_lqv'` but `lqv/typologies/cob_bottle_lqv.py` does not exist; the module lives at `lqv/subscene/cob_bottle_lqv.py`. Any consumer that does `importlib.import_module(f"lqv.typologies.{name}")` over `TYPOLOGIES` raises `ModuleNotFoundError` on the first entry. This is a broken public contract in the package's own `__init__`.
- `lqv/boq.py:119-123` — `_infer_qty` silently falls back to `(0.0, 'count')` when no recognized qty key is present. `lqv/typologies/bamboo_container_4pax.py:180-184` ships `'shipping_container_20ft'` with only `count` + `unit_cost_usd` and no `qty` / `m3` / `m2` / `kg` key. Net effect: the most expensive single line item in that typology can roll up as quantity 0.0 with no warning. The BoQ that backs the escritura deck PYG total may understate by the price of a 20-ft container.
- `lqv/site/terrain_dsl.py:414-427` — `validate_geo` checks "houses under water" by sampling `sample_height` at footprint corners *after* `_incise_channel` has already carved the heightfield. Validation runs against the post-carved surface that the houses sit on, so a creek that actually clips a foundation will not be flagged. This validator looks like it works and does not. If the audit is what we ship next to the escritura, it is lying.
- `lqv/typologies/hobbit_house.py:192-197` and `:293-296` — `try: bpy.ops.object.modifier_apply(...); except RuntimeError: pass`. Modifier-apply failures (bad context, no active object, modifier disabled) are swallowed; the export looks fine but the dome geometry silently retains a stack instead of being baked. Empty `pass` exception handlers are banned per global CLAUDE.md.
- `lqv/amenities/eco_retreat_modern_oasis.py:596,605-606,618,627-628` — four `except Exception:` blocks; lines 605-606 and 627-628 are bare `pass`. The amenity that fronts the master suite render can silently lose materials, modifiers, or boolean ops with zero log line. Banned per CLAUDE.md.
- `lqv/subscene/terrain_62ha.py:398-406` — `roof_faces` is assigned twice; the first assignment contains a literal typo placeholder tuple `(3, 7 - 0, 0, 0)` that is then overwritten. Dead code with an arithmetic-expression bug shipping at the bottom of the digital-twin renderer is a smell that says nobody re-read this file before T-DT was tagged at `4409dba`.
- `lqv/boq.py:42` (`USD_TO_PYG = 7300.0`) vs `scripts/build_escritura_deck.py:57` (`USD_TO_PYG = 7300`) — two sources of truth for the FX rate, different types (float vs int), used in the same deck pipeline. When the rate changes (it will: PYG has moved ~5% in any rolling 30-day window for years), exactly one will be updated and the deck will silently misprice.
- `scripts/build_escritura_deck.py:58` — `TODAY_ISO = "2026-06-13"` hardcoded. The deck builder embeds the date as a literal; any re-run after today stamps the wrong date on the escritura artefact and nobody will notice until the notary does.

## P1 — should fix before escritura

- `lqv/subscene/base.py:147` — `run(...)` default `clip_end=1000.0`. Parcel-scale sub-renders need 20 000 m or the HDRI is the only thing in frame (documented gotcha in `memory/MEMORY.md`). 19 driver files now hardcode `20000.0` to work around the unsafe default: `bamboo_beton_family_curved.py:59`, `bamboo_wigwam_lodge.py:73`, `floating_dining.py:48`, `italian_river_house_4pax.py:67`, `bamboo_beton_28.py:53`, `bamboo_boomhut_treehouse.py:36`, `bamboo_container_4pax.py:91`, `bamboo_beton_30.py:49`, `bamboo_beton_family_rectangular.py:116`, `boulder_cluster.py:144`, `bamboo_river_house.py:78`, `eco_pool.py:46`, `eco_retreat_modern_oasis.py:60`, `container_river_house.py:54`, `flora_jacaranda.py:63`, `flora_pachira.py:53`, `hobbit_house.py:78`, `flora_anthurium.py:63`, `italian_stone_small_v2.py:53`, `labrisa_lounge.py:60` + inlined again in `terrain_62ha.py:635`. Repeated magic number = next contributor forgets it and gets a black PNG.
- `lqv/site/terrain_dsl.py:352` — `cut` snap uses `target_z = float(min(corner_heights))`. Documented as "cut" but it raises low cells when the minimum corner is above existing terrain. It is "flatten to min", not "cut". The pad above the creek will silently get filled.
- `lqv/site/terrain_dsl.py:451` — creek slope test uses absolute `drop/length < 0.005` with no sign check. An uphill polyline (start_z < end_z) flags as "still water" with the same threshold, but a downhill creek with the same magnitude does not. Validator can pass a creek that flows uphill.
- `lqv/site/terrain_dsl.py:464-471` — footprint overlap test is AABB-vs-AABB on rotated rectangles. Two long rectangles on opposing diagonals at 45° will report overlap with no actual collision. False positives only, but the audit log will fill with noise that hides real issues.
- `lqv/site/terrain_dsl.py:474-485` — `tree_scatter` polygon vs house: scatter AABB vs house corner-AABB. Same shape problem as above; scatter polygons are arbitrary and this will fire constantly.
- `lqv/amenities/_grammar.py:128` — `mat_glass = _mat('lantern_paper_warm') or _mat('pv_glass') or _mat('glass')`. Three-level silent fallback chain. If `lantern_paper_warm` was renamed and `pv_glass` is also missing, lanterns render as plain glass with no log line. Same pattern is used elsewhere in the grammar (51, 63, 101, 133, 146, 186) for primitive ops — fragile fallbacks compound.
- `lqv/typologies/hobbit_house.py:140` — `_resolve(*keys)` silent material fallback chain. Identical anti-pattern; identical failure mode (silently wrong material on the hero hobbit-house render).
- `lqv/boq.py:197-207` — `per_material_rollup` collapses mixed-unit groups by setting `unit='mixed'` and `quantity=0.0`. The escritura deck calls this; any row whose material appears in two units (e.g. bamboo as `m` and `kg`) shows up with quantity 0, which a notary reading the BoQ will flag.
- `lqv/boq.py:295-297` — `by_module` iteration order comes from `pkgutil.iter_modules`, i.e. filesystem order. Across two machines (or even after a directory mtime nudge) the BoQ rows reorder. The escritura PDF should be byte-stable run-to-run.
- `lqv/boq.py:354` — pandoc PDF success check is `rc == 0 and size > 0`, not actual `%PDF-` magic bytes. A pandoc that emits a non-PDF (HTML fallback, error stub) passes the check. Cheap to make this real.
- `lqv/boq.py:168` — `except Exception as e:` (noqa BLE001). Broad swallow on the BoQ rollup hot path. At minimum log the exception type and the module being scanned, otherwise a typo in one typology's `MATERIAL_TAKEOFF` literal vanishes a whole module's rows.
- `lqv/subscene/base.py:36` — `_RUN_ID = os.environ.get('RENDER_RUN_ID') or datetime.now().strftime('%Y%m%d_%H%M%S')`. If the operator forgets to export `RENDER_RUN_ID`, every sub-render lands in its own timestamped folder and the A/B/C set is no longer co-located. Reasonable default for one-off, broken for batch.
- `lqv/subscene/base.py:120-125` — A/B/C variant exposure hardcoded (`-0.2 / 0.3 / 0.6`). A/B/C semantics are also embedded in HDRI rotation, sun strength, and elsewhere across driver files; no single source of truth. Renaming the variants in one place will desync the rest.
- `lqv/subscene/base.py:137-138` — `shutil.copy2` triple-writes every render (run folder + `latest/` + legacy flat). 3× disk + 3× I/O on every save. Cheap to make `latest` a symlink and drop legacy in favour of a one-time generator. Not a blocker, but it bloats `_archive/` and burns SSD writes.
- `lqv/subscene/terrain_62ha.py:158-161` — heightmap is loaded as a Python nested `list[list[float]]`. For a 4K heightmap that is ~64 MB of Python objects vs ~64 MB packed in numpy; the headless render machine will swap. Use `np.asarray(..., dtype=np.float32)` instead.
- `lqv/subscene/terrain_62ha.py:193-226` — local `_emission_material` / `_principled_material` bypass the `MAT` registry. Same project, two material-creation paths, registry drift inevitable.
- `lqv/subscene/terrain_62ha.py:462` — `h_jitter = 1.0 + 0.18 * ((i % 3) - 1)`. The docstring upstream claims deterministic-by-seed; `i % 3` is pseudo-RNG masquerading as determinism. Same anti-pattern at `lqv/amenities/_grammar.py:185` (`(i * 37) % 7`). If reviewers ever ask "why these positions", the honest answer is "because modulo".
- `lqv/materials/__init__.py:47` — `MAT['pool_water'] = glass.make_pool_water()` is a one-off insertion outside the `submodule.build(MAT)` fanout used by earth/wood/foliage/props. Two patterns side-by-side in 50 lines.
- `lqv/materials/glass.py:41` — `make_pool_water()` defined outside `build(MAT)`, the only material in the package that is not invoked through the standard entry point.
- `lqv/materials/glass.py:56` — `bsdf.inputs['Transmission Weight'].default_value = 1.0`. Blender 4.2 renamed `Transmission` → `Transmission Weight`; if the renderer is ever bumped to a build that reverts or renames again, this raises `KeyError` and pool/glass go solid. No defensive `try` / `.get`. (And no, swallowing it would be banned; the right answer is a feature probe at load.)
- `lqv/subscene/terrain_dsl.py:628-630` — wait, sorry: `lqv/site/terrain_dsl.py:628-630` — `cam.data.clip_end` is only increased if currently `< self.z_clip_end`. The documented gotcha says "set", not "max". Asymmetric vs the rest of the code base.
- `scripts/build_boq.py:26-69` — `_PermissiveNS` bpy stub. Clever for letting `MATERIAL_TAKEOFF` dict literals import outside Blender, but any module that does `isinstance(x, bpy.types.Object)` or sub-class checks at import time silently returns falsy. If a future typology drifts to import-time bpy use, BoQ will start dropping rows with no error.
- `scripts/build_boq.py:75-84` — `_install_materials_stub` is a no-op kept for back-compat. Dead-ish code in a script that ships the escritura BoQ is a 30-second cleanup that the next contributor will spend an hour wondering about.
- `scripts/render_queue.py:28` — `ROW_RE` regex is tightly coupled to the `STATUS.md` table format including backticks and exact spacing. Rename a column header and the queue-restart returns `0 pending` silently — the operator will think the render is done. Add a sanity assert: if STATUS.md is non-empty and `pending == 0` and `done == 0`, raise.
- `scripts/build_escritura_deck.py:22, 31-32` — reads `lqv.boq.collect_all()` AND parses `docs/boq/boq_rollup.md` text. Two parsers, two drift paths. Pick one.
- `scripts/build_escritura_deck.py:842 lines` — the deck builder is the largest single file in the repo. It is also one of two artefacts the notary will see. Should be the most reviewed; instead it is the most opaque.

## Roast highlights (one-liners)

- The package's own `__init__.py` advertises a typology that lives in a different package. `lqv/typologies/__init__.py:21` lies to importers about its own contents.
- The "houses-under-water" validator samples the surface that was carved to not flood the houses. Validation theatre on the escritura's biggest legal risk surface.
- The most expensive single line item in `bamboo_container_4pax` rolls up as quantity zero because `_infer_qty` couldn't find a key it recognised and chose silence.
- Two files, two FX rates, both spelled `USD_TO_PYG = 7300`. One is `7300`, the other is `7300.0`. They cannot both be right when the rate moves.
- `TODAY_ISO = "2026-06-13"` baked into the deck builder. The script knows what day it is. It was told.
- `clip_end = 20000.0` appears in 20 files because the default in `base.py` is 1000 and would render a black HDRI. The fix is one constant; the workaround is twenty.
- `terrain_62ha.py:398-406` has dead `roof_faces` code with the literal expression `(3, 7 - 0, 0, 0)` in it. Somebody was thinking about something else.
- Four `except Exception: pass` blocks in the hero amenity. The audit will not find what the code refused to tell it.
- `make_pool_water` is the one material in the registry that nobody told about the `build(MAT)` pattern.
